# dec-0002 可执行的本地运行编排闭环

## 状态

accepted

## 日期

2026-04-09

## 决策

`picasso-dev-skill` 自本次起把“环境通过后自动推进”从文字规则升级为脚本规则：

1. `deploy doctor` 必须显式校验本地运行编排入口
2. 冒烟脚本默认自动拉起后端和前端，并等待 ready
3. 如已配置登录入口，冒烟脚本默认自动获取 token
4. `stage-gate.py smoke` 默认真实执行冒烟脚本，并落执行日志
5. 脚本结束后默认自动停止本次拉起的本地服务

## 背景

- 用户希望这套 skill 在新机器、虚拟机、Claude/Codex 协作、单模型、国内外不同模型环境里都能稳定复现
- 如果“自动推进”只停留在 prompt 约束而不是脚本约束，最终会退化成手动启动和人工判断
- 当前 skill 已完成 local-only 边界和能力化 doctor，下一步必须补齐真正可执行的本地闭环

## 影响

- `.env` 不再只是路径配置入口，也成为本地联调与冒烟的统一编排入口
- `smoke gate` 的通过结论必须带真实执行证据
- 使用者在环境配置完成后，能够更稳定地得到“直接给出访问地址 / 账号密码 / 验证路径”的结果

## 代价与权衡

- `.env` 变量数会增加，首次接入配置成本略高
- 但能显著减少“模型理解不到位导致没启动服务、没真正联调、没真实冒烟”的隐性失败
- 如未来补浏览器自动化或页面级验证，应继续沿用当前 runtime-lib，而不是另起一套分散脚本
