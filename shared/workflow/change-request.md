# 需求变更工作流

## 目标

基于已有需求基线，明确变更范围、影响点和回归范围。

## 标准阶段

1. 读取旧需求基线
2. 明确本次变更点
3. 更新 `00-需求总览.md`
4. 更新 `*-需求文档.md`
5. 更新 `*-技术方案.md`
6. 若影响页面则更新 `*-UI交互设计规范.md`
7. 更新任务拆解与测试用例
8. 更新实现控制总表 / 页面接口验收总表
9. 开发与开发自测
10. 覆盖率、代码审查、冒烟、QA、验收

## 变更型额外要求

- 必须写清旧逻辑、本次变更点、回归范围
- 不允许只改代码不补差异说明
- 若变更影响公共接口、枚举、编码规则，必须回写到技术方案和公共规则引用处
