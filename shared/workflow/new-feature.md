# 新需求工作流

## 目标

把一个新需求从原始输入推进到可开发、可测试、可验收状态。

## 标准阶段

1. 创建需求目录
2. 归档原型 / 截图 / 参考资料
3. 产出 `00-需求总览.md`
4. 产出 `*-需求文档.md`
5. 产出 `*-任务分解.md`
6. 产出 `*-技术方案.md`
7. 产出 `*-UI交互设计规范.md`
8. 产出 `*-测试用例.md`
9. 产出 `*-实现控制总表.md`
10. 产出 `*-页面接口验收总表.md`
11. 编码与开发自测
12. 覆盖率与关键流程覆盖校验
13. 代码审查
14. 两关制冒烟测试
15. QA 与产品验收

## 强制 Gate 顺序

```text
需求文档
  ↓
任务分解卡点
  ↓
技术方案 / UI 规范
  ↓
编码开发
  ↓
开发自测卡点
  ↓
覆盖率卡点
  ↓
代码审查卡点
  ↓
冒烟测试卡点
  ↓
QA / 产品验收卡点
```

## 默认放行结论

- 每个阶段默认结论都是 `NEEDS_WORK`
- 没有证据的“通过”默认无效
- 同一任务连续失败 `3` 次，必须升级给 PM / 架构重拆
