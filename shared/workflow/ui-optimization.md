# UI 优化工作流

## 目标

对现有页面做视觉、布局、状态、端适配和交互优化，而不破坏既有业务逻辑。

## 标准阶段

1. 判断当前任务能力：`docs / dev / db / deploy`
2. 进入真实页面修改、接口联调、启动前先运行对应 `doctor.sh`
3. 记录现状问题
4. 产出 `*-UI交互设计规范.md`
5. 更新 `*-页面接口验收总表.md`
6. 明确改动边界
7. 更新任务拆解与测试用例
8. 执行页面优化
9. 开发自测、代码审查、体验回归
10. 如涉及真实接口联动，补冒烟、QA 验证、UI 验收和产品验收

## 默认边界

- 不改业务逻辑和接口语义
- 可改布局、样式、信息层级、状态表达、交互细节
- 即使只做 UI，也不能跳过证据采集与页面验收闭环
- `doctor` 未通过前，不进入真实页面编码或联调
