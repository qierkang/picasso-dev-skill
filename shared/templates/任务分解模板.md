## 2.5 强制质量 Gate

| Gate | 触发时机 | 必要输入 | 必要输出 | 通过标准 | 不通过动作 |
|------|---------|---------|---------|---------|-----------|
| 任务分解卡点 | PRD / 需求文档完成后 | 需求文档、技术/UI基线 | `任务分解.md` | 任务粒度清晰、证据要求明确 | 回 PM 重拆 |
| 开发自测卡点 | 编码完成后 | 代码变更、本地环境(local/dev) | `开发放行报告.md` | 0 阻塞项 | 回开发重测 |
| 覆盖率卡点 | 自测后 / review 前 | 测试结果、覆盖数据 | `覆盖率报告.md`、`前端关键流程覆盖清单.md` | 后端覆盖率 `>= 90%`，前端关键流程齐全 | 回开发补测 |
| 代码审查卡点 | 覆盖率通过后 | 代码 diff、编译结果 | `代码审查报告.md` | 0 `P0 / 阻塞` | 回开发修复 |
| 冒烟测试卡点 | review 通过后 | 本地环境(local/dev)、冒烟脚本 | `冒烟测试报告.md` | 两关制结果 `100% 通过` | 回开发 / 配置修复 |
| QA 验证卡点 | smoke 通过后 | 用例、接口、冒烟结果 | `QA验收报告.md` | 功能与边界场景无阻塞结论 | 回开发 |
| UI 验收卡点 | QA 通过后 | 页面、交互、UI 规范 | `UI验收报告.md` | 页面还原度、交互符合度无阻塞结论 | 回前端 / UI |
| 产品验收卡点 | UI 验收通过后 | 业务基线、页面结果、QA 结果 | `产品验收报告.md` | 业务与流程符合度无阻塞结论 | 回开发 / PM |

## 5. Gate 条件

- 无需求基线，不进入技术方案
- 无技术方案，不进入编码
- 无 UI 规范，不进入前端 / 小程序 / App 页面开发
- 无开发放行报告，不进入 review
- 无覆盖率报告与前端关键流程覆盖清单，不进入 review
- 无代码审查与冒烟结果，不宣称可验收
- 无 QA 验证结果，不宣称可进入 UI / 产品验收
- 无 UI 验收结果，不宣称可进入产品验收
- 默认结论不是 `READY`，而是 `NEEDS_WORK`
