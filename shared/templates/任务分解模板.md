## 3. 角色分工

| 角色 | 主要职责 | 不负责事项 |
|------|----------|-----------|
| `picasso-dev` | 总控、阶段推进、卡点管理 | 不直接跳过需求和方案写代码 |
| `picasso-dev-task` | 服务端方案、任务推进、公共依赖确认 | 不直接定稿 UI |
| `picasso-dev-ui` | UI 规范、三端差异、交互约束 | 不定义服务端接口语义 |
| `开发线程` | 实现、编译、测试、问题修复 | 不擅自改需求边界 |
| `QA / 产品` | 验收与缺陷回流 | 不替代开发自测 |

## 4. 阶段产出物清单

- 需求阶段：`需求文档`、`需求总览`
- 方案阶段：`技术方案`、`UI交互设计规范`
- 控制阶段：`实现控制总表`、`页面接口验收总表`
- 执行阶段：`开发放行报告`、`代码审查报告`
- 测试阶段：`冒烟测试报告`、`QA与产品验收报告`

## 5. Gate 条件

- 无需求基线，不进入技术方案
- 无技术方案，不进入编码
- 无 UI 规范，不进入前端 / 小程序 / App 页面开发
- 无开发放行报告，不进入 QA
- 无代码审查与冒烟结果，不宣称可验收

## 6. Dev↔QA 回环策略

- QA 提出问题后，先回写问题清单，再回到开发阶段修复
- 修复完成后，必须补新的验证证据
- 不允许只口头回复“已修复”，必须落报告
