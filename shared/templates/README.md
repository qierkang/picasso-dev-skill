# 模板目录说明

## 1. 目录用途

本目录用于定义 `picasso-dev-skill` 默认生成的需求产物模板。

当前采用两条规则：

1. 需求目录使用英文代号，例如 `maintenance-plan`
2. 目录内文档优先使用中文命名，方便团队直接查看

## 2. 默认主产物

新需求目录默认生成以下 `15` 个主产物：

1. `00-需求总览.md`
2. `manifest.json`
3. `功能名称-需求文档.md`
4. `功能名称-任务分解.md`
5. `功能名称-技术方案.md`
6. `功能名称-UI交互设计规范.md`
7. `功能名称-测试用例.md`
8. `功能名称-实现控制总表.md`
9. `功能名称-页面接口验收总表.md`
10. `功能名称-开发放行报告.md`
11. `功能名称-代码审查报告.md`
12. `功能名称-冒烟测试脚本.sh`
13. `功能名称-冒烟测试报告.md`
14. `功能名称-QA与产品验收报告.md`
15. `stage-status.json`

## 3. 使用原则

- 默认先生成主产物，再按需要补充 `legacy/` 或专项文档
- 默认不再使用日期作为文件名前缀
- 公共规则优先从 `../references/requirements/` 引用
- `需求文档模板.md` 顶部已内置公共说明块：
  - `术语定义`
  - `字段来源类型说明`
  - `文档编写强约束`
  - `通用系统字段快捷块`
  - `系统字段交互规格模板`
