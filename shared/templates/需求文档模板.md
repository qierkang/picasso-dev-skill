## 阅读约定

### 术语定义

- `搜索筛选`：表示页面最头部的查询区域。
- `数据展示`：表示列表或详情中的数据展示区域。
- `操作按钮`：表示当前页面所有可以操作的按钮。
- `按钮展示逻辑`：表示每个按钮在不同状态、权限下的展示与可用逻辑。
- `表单定义`：表示新增、编辑、查看场景下字段的显示、只读、必填、默认值、联动规则。
- `业务规则`：表示细节规则、特殊边界、联动规则和限制条件。

### 字段来源类型说明

| 类型 | 含义 | 典型场景 | 写法要求 |
|------|------|---------|---------|
| 用户输入 | 用户主动录入或选择 | 写清组件、必填、校验 |
| 系统默认带出 | 登录人/公司/租户自动回填 | 公司编码、公司名称 | 写清默认来源、是否允许修改 |
| 关联选择自动带出 | 选 A 后自动带出 B/C | 部门负责人、岗位概述 | 写清触发字段、回填字段、是否只读 |
| 系统计算/生成 | 保存前或展示时自动生成 | 不得写成手工输入 |
| 隐藏字段 | 页面不展示但落库或计算需要 | 审批开关、内部标记 | 写清是否前端透传或后端补齐 |
| 查看态只读 | 某些场景可编辑，查看态只读 | 审批状态、系统带出字段 | 写清新增、编辑、查看差异 |

### 文档编写强约束

- 系统已有公共字段、组织字段、人员字段、岗位字段、公司字段，必须写清获取方式，不能只写“数据源”或“用户填写”。
- 如果真实交互是“选择器 + 自动带出编码/名称/负责人”，必须在表单定义中明确写出，不得拆成多个“自由输入字段”。
- 如果真实交互是“弹窗选择器”，必须写清：组件类型、是否弹窗、是否带搜索区、单选或多选、回填字段、展示格式、默认值来源、是否允许改选。
- 如果某字段最终落库，但前端不应传值，必须明确写为“系统生成”或“后端补齐”。
- 如果某字段是展示拼接值，必须标注为“系统计算展示”或“冗余展示字段”。
- 当文档中出现“工号+姓名”“工号和姓名”“人员编码+姓名”等描述时，统一落为显示格式 `工号-姓名`。
- 当文档中出现“公司编码+公司名称”“公司编码和公司名称”等描述时，统一落为显示格式 `公司编码-公司名称`。
- 所有 `申请人` 字段统一使用 `弹窗选择器`，不再写成“默认当前登录人”或“普通输入框”。
- 表单中的所有 `Select` 组件均默认支持搜索；如果不是，必须显式说明原因。
- 原始内容如果已经是表格，优化后优先保留表格结构
- 需求文档要优先写清“数据来源、展示格式、是否可编辑、何时回写、状态如何流转”，不要只描述页面长什么样。
- 涉及联动链路时，必须写清来源模块、目标模块、触发时机、写入字段、状态变化。
- 涉及审批流时，至少写清“未接审批流”和“已接审批流”两种分支逻辑。
- 涉及详情页时，必须补充 `详情页按钮行为`，至少写清新增、编辑、查看三种进入方式下的按钮展示差异。
- 涉及主表、子表场景时，必须明确标注 `主表`、`子表`，并写清是否支持多条维护。

### 通用系统字段快捷块

| 字段组 | 推荐交互类型 | 默认来源策略 | 用户是否可编辑 | 常见备注写法 |
|------|-------------|-------------|---------------|-------------|
| 申请人 | 弹窗选择器 | 无默认值，需主动选择 | 是 | 点击输入框或放大镜打开用户弹窗，单选后回填 `工号-姓名` |
| 所属公司 | 关联自动带出 | 根据岗位或登录租户回填 | 否 | 只读展示，保存时后端校验一致性 |
| 部门 | TreeSelect / 部门弹窗 | 默认当前登录人所属部门，或通过选择器切换 | 是 | 回填部门编码、部门名称、部门负责人 |
| 部门负责人 | 关联自动带出 | 根据部门自动回填 | 否 | 选部门后自动带出 |
| 岗位 | Select / 岗位弹窗 | 默认当前登录人岗位，或通过岗位选择器切换 | 是 | 回填岗位编码、岗位名称、岗位概述、所属公司 |
| 岗位概述 | 关联自动带出 | 根据岗位自动回填 | 否 | 只读 |
| 单号 / 业务编码 | 系统计算 / 生成 | 编码规则或内部编码服务 | 否 | 前端只读展示，后端生成 |
| 审批状态 | 隐藏字段 / 查看态只读 | 系统默认值或流程状态 | 否 | 未接审批流时写清默认值与保存后状态 |
