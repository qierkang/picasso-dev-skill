# Web 页面模式

## 适用范围

- PC Web
- H5 Web 管理页
- Dashboard
- 列表页 / 详情页 / 表单页

## 常用模式

### 1. 列表 + 右侧详情

适用：

- 查询频繁
- 明细字段较多
- 切换项目需要快

优点：

- 不打断浏览流
- 适合运营人员连续处理

### 2. 列表 + 弹窗编辑

适用：

- 字段量中等
- 新增 / 编辑是短动作

避免：

- 把长表单塞进小弹窗

### 3. 独立详情页 + Tab

适用：

- 主子表
- 多模块详情
- 生命周期长的业务对象

建议：

- 顶部主信息固定
- Tab 内只承载一类子域

### 4. 看板式卡片列表

适用：

- 项目、任务、工单、活动
- 需要快速看状态、负责人、进度

禁忌：

- 卡片字段过多导致像表格但更难扫

## Web 信息层级规则

1. 首屏只保留当前最关键的查询和主操作
2. 筛选区与结果区分离
3. 批量操作必须和单条操作分层
4. 危险操作不要和主按钮并列抢主次
5. 空态要包含下一步动作

## Web 组件建议

- 查询条件多时优先折叠次要筛选
- 表格行操作超过 3 个时，用更多菜单收口
- 长文本字段使用 tooltip 或详情展开
- 状态标签必须颜色和文案双重表达

## 页面实现提醒

- 响应式不是简单缩放
- 表头、筛选、主按钮优先考虑键盘和鼠标效率
- 密集业务页优先保证扫描效率，再谈视觉装饰
