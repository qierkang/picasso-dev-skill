# 设计参考索引

本目录承接 `picasso-dev-skill` 的设计知识底座。

定位不是“再造一套外部设计框架”，而是把后续需求开发最常用、最容易缺失的 UI 设计与多端交互规则收进仓库，供 `picasso-dev-ui` 按需加载。

## 使用顺序

1. 先读 `设计风格方向库.md`
2. 再读 `设计Token基线.md`
3. 按终端继续读：
   - PC Web：`Web页面模式.md`
   - iOS / SwiftUI：`iOS-SwiftUI规范.md`
   - Android：`Android-Material3规范.md`
   - 微信小程序：`微信小程序交互规范.md`
4. 涉及动画、反馈、加载、异步行为时，再读 `动效与反馈规范.md`
5. 涉及响应式、可访问性、触控热区、读屏时，再读 `可访问性与响应式规范.md`

若外部 `ui-ux-pro-max` 不可识别，默认只允许使用本目录完成设计兜底；即使运行端还能识别 `frontend-design` 等通用设计 skill，也不得把它们当成默认回退路径。

## 设计输出最低要求

UI 设计文档至少要明确：

1. 风格方向
2. 信息层级
3. token 基线
4. 组件映射
5. 动效与反馈
6. 平台专项约束
7. 验收口径

## 外部思路来源

- 通用设计知识底座：参考 `ui-ux-pro-max-skill` 的“风格库 + 多栈映射 + UX 规则”思路做收敛
- iOS 专项：参考 `SwiftUI-Agent-Skill` 的“SwiftUI / HIG / 可访问性 / 审查”思路做收敛

这里只吸收高价值设计方法，不把外部 skill 作为运行时依赖。
