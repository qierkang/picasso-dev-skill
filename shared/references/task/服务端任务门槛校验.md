# 服务端任务门槛校验

## 支持批量校验

当用户说“校验任务”或“校验所有任务”时，可以扫描 `incoming/` 下所有任务后一次性反馈。

## 必填项

以下项目缺一不可：

- 任务名称
- 所属子产品
- 优先级：`P0 / P1 / P2`
- 代码分支
- 概述
- 功能描述

## 可选但要提示

缺失时不阻塞，但必须提示用户确认：

- 数据实体
- 数据权限 / 状态控制
- 关联子产品
- 原型图 / 交互参考

## 校验动作

1. 检查概述是否与已有 active 任务重复
2. 检查所属子产品是否已有模块映射
3. 检查代码分支是否给出明确值
4. 检查是否缺少影响设计的关键信息

## 结果输出格式

```text
校验结果：✅ 通过 / ❌ 不通过
- <任务名>：✅ 通过
- <任务名>：❌ 缺少代码分支
```
