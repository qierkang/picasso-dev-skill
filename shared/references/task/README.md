# Picasso 服务端任务规则

这组文档吸收自旧版 `picasso-task`，用于补齐 Picasso 服务端任务推进时的项目专属规则。

按需读取：

- 创建服务端任务输入时：`服务端任务输入模板.md`
- 查看任务与源码目录布局时：`服务端任务路径布局.md`
- 校验任务是否可开启时：`服务端任务门槛校验.md`
- 设计服务端方案时：`服务端设计约束.md`
- 维护 active 任务状态时：`服务端任务状态管理.md`
- 归档、恢复、撤销任务时：`服务端任务归档恢复与撤销.md`
- 处理服务端需求变更时：`服务端需求变更流程.md`
- 根据用户说法判断动作时：`服务端快捷触发语义.md`
- 组织 Claude + Codex 或单模型编码协作时：`服务端 Codex 协作规范.md`
- 判断子产品与模块归属时：`子产品与模块映射.md`

这些文档是 Picasso 项目专属规则，不是通用 skill 模板。

进入真实编码前，还必须确认：

- 已产出 `任务分解.md`
- 已准备 `Codex CLI 标准执行输入`
- 已明确 `开发放行报告 / 覆盖率报告 / 前端关键流程覆盖清单 / 代码审查报告 / 冒烟测试报告` 的回填责任

## 当前覆盖范围

这组规则当前覆盖服务端任务的完整生命周期：

1. `incoming` 新任务创建与门槛校验
2. `active` 任务设计、任务拆解、状态回写、分批开发
3. 需求变更分级、影响分析、文档回退与回流
4. `archived` 任务归档、恢复、补产物与撤销

## 最低执行原则

- 开始任何新的 `openspec` 文档前，先读取 `openspec/specs/`
- `status.md` 必须由主流程维护，不交给编码模型定稿
- 归档必须使用 `openspec archive`，不要用手工 `mv` 冒充归档
- 变更影响未评估前，不允许直接改代码
