# 服务端设计约束

这份规则用于补齐 Picasso 服务端设计阶段的项目特有要求。

## 设计前必须确认

以下信息缺失时，先向用户确认，不要擅自假设：

- 所属模块
- 表名
- 字段类型 / 长度
- 枚举值
- 接口路径
- 权限标识
- 编码生成方式
- 删除策略

设计前还必须先扫描相关源码：

- Controller
- Service / ServiceImpl
- Mapper / XML
- DO / BO / VO

不要只看文档或历史记忆推断现有实现。

## 数据库约束

- 表名使用蛇形命名
- 主键使用 PostgreSQL 自增
- DDL 不写唯一索引，由业务层保证唯一性
- DDL 不写 CHECK 约束
- DDL 必须同时提供 schema 与 rollback
- 字典 / 枚举映射必须写清

## 组件复用约束

- 先复用已有组件与已有 API，再考虑新增
- 判断“已有实现”时只看源码，不看 `target/`、`build/` 等编译产物
- 优先复用已有数据库约定、通用组件、已有编码场景和异常处理模式

## 接口设计约束

- design 中默认写服务端开发接口路径，不直接混入运行时统一前缀
- 必须标注：接口路径、是否匿名、权限标识、接口端
- 涉及异常处理、导入导出、BPM、文件读写、外部 API 时，必须补异常分类和日志策略

## 补充产物约束

- design 确认后，先生成 `MENU_DATA.md`
- 涉及字典字段时，同时生成 `DICT_DATA.md`
- 未生成 `MENU_DATA.md / DICT_DATA.md` 前，不进入 `tasks.md`

## 主子表约束

如涉及主子表，必须写清：

- 关联字段
- 级联规则
- 删除策略
- 引用校验规则
