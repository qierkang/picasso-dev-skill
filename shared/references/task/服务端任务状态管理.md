# 服务端任务状态管理

这份规则用于约束 `${PICASSO_SERVER_TASK_DIR}/active/<任务名>/status.md` 的维护方式。

## 文档确认顺序

`proposal.md`、`specs/`、`design.md` 完成后都必须经用户确认，才能进入下一步。

```text
proposal [待确认]
  ↓ 用户确认
specs [待确认]
  ↓ 用户确认
design [待确认]
  ↓ 用户确认
MENU_DATA.md / DICT_DATA.md [自动生成]
  ↓
tasks.md [自动生成]
```

## 必须更新的时机

以下节点必须回写 `status.md`：

1. 每个文档生成时
2. 每个文档确认时
3. `tasks.md` 生成后
4. 每一批 Codex 开发任务结束后
5. design 发生调整或需求变更时
6. 进入归档前
7. 归档完成后

## 强制约束

- 不得跳过 `tasks.md` 与 `status.md` 回写，直接进入下一批编码
- 编码模型可以提供实现事实和验证结果，但 `status.md` 由主流程最终维护
- design 变更后，必须先更新 `design.md` 与 `status.md`，再调整 `tasks.md`

`status.md` 至少包含以下区块：

```markdown

- 变更描述：
```

## 归档触发规则

- `tasks.md` 最后一个任务完成后，必须主动提示用户是否归档
- 归档前必须确认 `status.md` 内容完整且与真实进度一致
- 归档前必须运行 `openspec validate --change <任务名>`
