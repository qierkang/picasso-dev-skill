# 服务端 Codex 协作规范

## 协作边界

Codex 或编码模型只负责：

- 代码实现
- 编译验证
- 测试执行
- 必要的 SQL / 配置 / 文件落地
- 输出实现事实、接口参数结构、验证结果、变更摘要

主流程负责：

- `status.md` 更新
- 阶段状态推进
- 接口联调文档最终定稿
- 是否归档和何时归档的判断

## 禁止事项

- 不要让编码模型直接回写 `status.md`
- 不要让编码模型直接定稿联调文档 / API 文档
- 不要把流程性文档维护和代码实现混在一次编码执行里

## 执行前提

1. 先确认变更方案
2. 先更新 design / tasks，再进入代码执行
3. 开始任何新的 openspec 文档前，先回 `master` 更新基线，再切任务分支

## 增强模式

当环境使用 Claude + Codex 时：

- Claude 负责需求、方案、设计、任务拆解和复核
- Codex 负责代码实现、编译、测试和修复

## 单模型模式

当只有一个模型时：

- 仍然遵守同样的边界
- 只是由同一个模型按阶段切换角色推进
