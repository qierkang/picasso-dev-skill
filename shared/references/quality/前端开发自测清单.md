# 前端开发自测清单

> 前端 / 小程序 / App 页面修改完成后，必须先自测并回填到 `开发放行报告` 或对应证据中，再进入 review。

## 通用检查项

| # | 检查项 | 方法 |
|---|--------|------|
| 1 | 页面加载无白屏 | 直接访问页面，确认正常渲染 |
| 2 | Console 无红色错误 | 打开浏览器开发者工具检查 |
| 3 | 表格 / 列表正常显示 | 确认列名、数据、分页、空状态 |
| 4 | 查询功能生效 | 输入条件后触发查询并校验结果 |
| 5 | 重置功能生效 | 重置后查询条件恢复默认 |
| 6 | 新增流程可执行 | 打开表单、填写、提交、校验提示 |
| 7 | 编辑流程可执行 | 编辑回显正确、保存后列表刷新 |
| 8 | 查看态只读 | 查看页面 / 弹窗不能误编辑 |
| 9 | 删除 / 关键操作反馈明确 | 成功、失败、阻塞提示清晰 |
| 10 | 异常状态可见 | 空态、加载态、错误态可见 |

## 强制输出格式

- 每项必须标记 `✅` 或 `❌ + 原因`
- 发现 `❌` 不能进入 review
- 发现问题后修复，必须补新的证据
