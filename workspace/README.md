# workspace 说明

本目录用于存放运行中的需求产物、归档、缓存和临时文件。

## 目录用途

- `requests/`：进行中的需求目录
- `archives/`：已完成或已冻结的需求目录
- `server-tasks/`：服务端任务协同目录
- `cache/`：运行时缓存
- `tmp/`：临时文件

## 规则

1. 需求目录统一使用英文代号，例如 `maintenance-plan`。
2. 日期只保留在 `manifest.json` 和文档元信息中，不进入目录名。
3. 本目录默认不提交到 Git。
4. 需求目录内部文档优先使用中文命名，例如 `保养方案-需求文档.md`。
