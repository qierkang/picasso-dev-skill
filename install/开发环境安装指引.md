# Picasso Dev Skill 开发环境安装指引

## 目标

让新机器在真正进入 Picasso 开发前，先把需要的本地环境补齐。

默认原则：

1. 纯文档 / PM / 方案任务可以只用 `docs` 能力，不强依赖开发工具链
2. 一旦进入编码、联调、SQL、冒烟、发布准备，必须先通过 `doctor.sh`
3. 当前 skill 只允许 `local` / `dev` 视为本地环境
4. `test` / `uat` / `prod` 只用于识别边界，不允许连接、查询、部署
5. `doctor` 没通过时，不进入真实开发；先修环境，再重跑同一能力直到通过
6. 进入真实开发后默认自主推进，不反复询问是否继续；只有环境无法补齐时才升级给使用者

## 能力判断

进入任务前，先判断本次只需要哪类能力：

- `docs`：需求文档、技术方案、UI 规范、任务拆解、培训演示
- `dev`：后端 / PC / 小程序 / App 编码与本地联调
- `db`：本地数据库查询、SQL 校验、数据回看
- `deploy`：本地启动、冒烟、本地发布准备

如果同一线程同时涉及多类动作，按需要依次补齐：

1. 先过 `docs`
2. 再过 `dev`
3. 涉及 SQL 再过 `db`
4. 涉及启动或冒烟再过 `deploy`

## 失败处理闭环

标准闭环如下：

1. 运行对应能力的 `doctor.sh`
2. 逐条处理 `FAIL`
3. 重新运行同一能力的 `doctor.sh`
4. 直到该能力无 `FAIL`
5. 再继续后续开发动作

不要在环境未通过时直接跳过。

## 推荐顺序

1. 克隆仓库并创建 `.env`
2. 执行 `bash install/setup.sh`
3. 先执行 `bash install/doctor.sh --capability dev`
4. 如需本地数据库查询 / SQL 校验，再执行 `bash install/doctor.sh --capability db`
5. 如需本地启动、冒烟、发布准备，再执行 `bash install/doctor.sh --capability deploy`
6. 所有需要的能力通过后，再进入真实开发

## docs 能力

适用场景：

- 需求文档
- 技术方案
- UI 规范
- 任务分解
- 培训演示

最低要求：

- `git`
- `python3`
- `.env`
- `workspace/` 可写

命令：

```bash
bash install/doctor.sh --capability docs
```

## dev 能力

适用场景：

- 后端编码
- PC 前端编码
- 小程序编码
- App 编码
- 本地联调

最少要求：

- 后端：`java 21+`、`maven` 或 `mvnw`
- 前端 / 小程序：`node 20.12.0+`、`pnpm 10.14.0+`
- App：`flutter`

命令：

```bash
bash install/doctor.sh --capability dev
```

### macOS 常见安装命令

```bash
brew install openjdk@21
brew install maven
brew install node
npm install -g pnpm
```

如涉及 Flutter：

```bash
brew install --cask flutter
```

## db 能力

适用场景：

- 查询本地数据库
- 校验 SQL
- 回看本地数据状态

额外要求：

- 本地数据库 Host 必须是 `127.0.0.1` / `localhost` / `::1`
- 安装 `psql`

命令：

```bash
bash install/doctor.sh --capability db
```

### macOS 安装 psql

```bash
brew install libpq
echo 'export PATH="/opt/homebrew/opt/libpq/bin:$PATH"' >> ~/.zprofile
source ~/.zprofile
```

如果你主要通过交互式 shell 工作，也可以同步写入 `~/.zshrc`。

### Ubuntu / Debian 安装 psql

```bash
sudo apt-get install postgresql-client
```

## deploy 能力

适用场景：

- 本地启动服务
- 本地冒烟
- 本地发布准备

边界：

- 只允许 `local` / `dev`
- 不允许连 `test` / `uat` / `prod`
- 不允许配置测试库、测试 SSH、远程发布入口

命令：

```bash
bash install/doctor.sh --capability deploy
```

## 环境边界识别

Picasso 服务端仓库里，以下 profile 视为边界依据：

- `application-local.yaml`
- `application-dev.yaml`
- `application-test.yaml`
- `application-uat.yaml`
- `application-prod.yaml`

skill 在设计上默认只允许：

- `local`
- `dev`

其余 profile 即使文件存在，也只用于提醒，不作为允许连接和部署的依据。
