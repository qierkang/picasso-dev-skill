# Picasso 配置入口说明

## 唯一配置入口

新机器接手时，默认只需要修改仓库根目录的 `.env`。

不要优先去改 `profile.yaml`、`repo-map.md`、`README.md` 里的路径描述。

## 必改项

以下配置改完后，新的开发人员就能开始使用：

- `PICASSO_ENV_BOUNDARY_MODE`
- `PICASSO_ALLOWED_RUNTIME_PROFILES`
- `PICASSO_FORBIDDEN_RUNTIME_PROFILES`
- `PICASSO_SERVER_CODE_DIR`
- `PICASSO_FRONT_CODE_DIR`
- `PICASSO_SERVER_TASK_DIR`
- `PICASSO_WORKSPACE_DIR`
- `PICASSO_BACKEND_START_WORKDIR`
- `PICASSO_BACKEND_START_CMD`
- `PICASSO_BACKEND_BASE_URL`
- `PICASSO_BACKEND_HEALTH_URL`
- `PICASSO_FRONTEND_START_WORKDIR`
- `PICASSO_FRONTEND_START_CMD`
- `PICASSO_FRONTEND_READY_URL`
- `PICASSO_AUTO_START_LOCAL_SERVICES`
- `PICASSO_AUTO_STOP_LOCAL_SERVICES`
- `PICASSO_RUNTIME_READY_TIMEOUT_SEC`

如果项目还包含对应端，再补：

- `PICASSO_MINIAPP_CODE_DIR`
- `PICASSO_MOBILE_APP_CODE_DIR`

如果希望冒烟脚本自动拿登录态，再补：

- `PICASSO_AUTO_FETCH_ACCESS_TOKEN`
- `PICASSO_LOGIN_URL`
- `PICASSO_LOGIN_METHOD`
- `PICASSO_LOGIN_USERNAME`
- `PICASSO_LOGIN_PASSWORD`
- `PICASSO_LOGIN_TENANT_ID`
- `PICASSO_LOGIN_TENANT_HEADER`
- `PICASSO_LOGIN_REQUEST_BODY_TEMPLATE`
- `PICASSO_LOGIN_TOKEN_PATH`

## 平台接入项

只配置你实际使用的平台技能目录：

- `OPENCLAW_SKILLS_DIR`
- `CLAUDE_SKILLS_DIR`
- `CODEX_SKILLS_DIR`

如果当前只用一个模型或一个平台，未使用的平台可以留空。

## 推荐接手顺序

1. `cp .env.example .env`
2. 修改 `.env`
3. 执行 `bash install/setup.sh`
4. 执行 `bash install/doctor.sh --capability dev`
5. 执行 `bash install/doctor.sh --capability deploy`
6. 如需同步 skill，再执行 `bash install/sync.sh`

## 校验原则

- `doctor.sh` 必须通过必填项校验后，再开始真实需求开发
- 如 `.env` 中仍保留 `/path/to/...` 这类占位值，视为未配置完成
- 当前 skill 固定是 `local-only` 模式：`local / dev` 视为本地，`test / uat / prod` 禁止连接和部署
- 如只做文档、方案、UI 等非编码任务，可按需使用 `bash install/doctor.sh --capability docs`
- 如要让“启动后端 → 启动前端 → 自动联调 → 自动冒烟 → 自动收尾”真正全自动，必须把运行编排配置补齐并通过 `deploy doctor`
