# Picasso 执行模式

## 默认模式：单模型独立执行

适用场景：

- 只有一个模型 CLI 可用
- 当前环境不具备 Claude + Codex 协作条件
- 需要最稳的可复制方案

要求：

- 同一模型按 `picasso-dev` 流程完整推进
- 逻辑角色可以拆分，但不依赖真实多 agent
- 进入开发前先自动体检环境；环境未通过时先修环境，不继续推进编码
- `doctor` 通过后默认继续执行本地启动、联调、冒烟、收尾，不反复询问是否继续
- 若 `.env` 已配置运行编排入口，应默认自动拉起后端与前端、自动登录拿 token、自动收尾

## 增强模式：Claude + Codex

适用场景：

- Claude 负责需求、方案、流程调度
- Codex 负责代码实现、验证、审查辅助

说明：

- `codex-plugin-cc` 是增强选项，不是必需依赖
- 没有该插件时，仍可用手动方式分工执行
- 不因为协作模式变化而跳过本地环境预检与 local/dev 边界
- 不因为双模型协作而把自动联调、自动冒烟回退成人工操作；只要环境具备，就默认跑完

## 平台模式：OpenClaw 或其他平台

适用场景：

- 团队已有统一 agent 平台
- 希望所有模型都只围绕当前仓库这一份 canonical skill 源目录使用

说明：

- 不再把 `~/.openclaw/skills`、`~/.claude/skills`、`~/.codex/skills` 作为副本安装前提
- 统一以当前仓库路径为准
- 平台切换不改变环境边界：开发、自测、SQL、部署仍只允许 local/dev
- 平台切换也不改变默认执行风格：先 `doctor`，再自动推进本地闭环；只有真实阻塞才升级给使用者
