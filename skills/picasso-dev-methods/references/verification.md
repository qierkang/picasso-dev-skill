# Verification Before Completion

## 目标

任何“已完成”“已修复”“可以提测”的结论，都必须有真实验证支撑。

## 默认顺序

1. 文档与状态回填
2. 编译 / 构建通过
3. 自动化测试通过
4. 开发自测通过
5. 覆盖率与关键流程清单补齐
6. 冒烟脚本真实执行通过
7. `stage-gate.py` 对应阶段通过

## 证据类型

- 命令输出
- 测试结果
- 冒烟结果
- 页面访问 / 接口调用记录
- 验收报告中的结论与缺陷项

## Picasso 硬规则

- 没有证据，不算完成
- 没过 gate，不算放行
- 默认结论先写 `NEEDS_WORK`，不是 `READY`

## 输出要求

最终汇报时，优先给出：

- 做了什么
- 怎么验证的
- 哪些还没验证
- 当前阻塞或残余风险
