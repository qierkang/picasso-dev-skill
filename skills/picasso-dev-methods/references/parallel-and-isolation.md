# Parallel And Isolation

## 目标

在多代理或多人协同时，既提速，也避免互相踩文件、踩环境、踩状态。

## 默认原则

1. 主入口只有一个：`picasso-dev`
2. 并行只拆独立子问题，不拆共享写入面过大的任务
3. 能分目录、分模块、分阶段时再并行
4. 共用环境时，优先串行关键步骤，避免相互覆盖

## 适合并行的场景

- 文档补齐与代码实现分开
- 服务端与 UI 方案分开
- 多个互不重叠的页面 / 模块改动
- 审查 / 验证与非冲突实现并行

## 需要隔离的场景

- 大范围重构
- 同一文件多人同时修改
- 需要独立分支或 worktree 做实验
- 本地运行编排会占端口、写缓存或修改共享配置

## Picasso 落地要求

- 并行前先约定写入边界
- 必要时使用独立 worktree 或独立需求目录
- 最终仍需回收到同一套阶段产物和 gate 结果中

## 反模式

- 为了并行而并行
- 一个需求拆出多套主流程
- 每个执行端各写一套规则，最后互相不兼容
